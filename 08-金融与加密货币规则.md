# 金融与加密货币规则

## 零、项目重写原则

1. **不依赖外部仓库**：所有规则在本项目内自行维护创建，不依赖第三方规则仓库。
2. **按组审查确认**：每一组规则先确认设计，再进入完整配置。
3. **最小可用原则**：只保留必要的规则，不盲目复制大量个人偏好列表。

---

## 一、策略组设计

### 1.1 金融统一策略组

所有金融与加密货币相关服务统一走 `金融` 策略组：

```yaml
proxy-groups:
  - name: 金融
    type: select
    proxies:
      - 新加坡-故转    # 首选：新加坡节点，金融服务友好
      - 所有-自动
      - 香港-故转      # 备选：香港节点
      - 美国-故转      # 备选：美国节点，适合美股相关服务
      # 其他节点...
```

**推荐默认选「新加坡-故转」**：对大多数国际金融服务友好，稳定性好。

## 四、注意事项

### 4.1 金融交易安全建议

金融交易对安全性和稳定性要求很高：

1. **尽量使用固定节点**，不要频繁切换 IP，避免触发风控
2. **不要使用公共免费节点**进行金融交易，存在安全风险
3. **建议开启 UDP 转发**，提升连接稳定性
4. 如果遇到风控或无法登录，尝试切换节点或清理浏览器缓存

### 4.2 节点地区选择建议

| 服务类型 | 推荐节点地区 |
|----------|-------------|
| 加密货币交易所 | 新加坡 / 香港 / 日本 / 美国 |
| 美股券商 | 美国 / 新加坡 |
| 港股券商 | 香港 / 新加坡 |
| 国际支付 | 新加坡 / 香港 / 美国 |

### 4.3 国内金融服务

国内金融服务（如国内银行、A股券商、支付宝、微信支付等）：
- **不需要走代理**，直连即可
- 走代理反而可能触发风控或无法访问
- GEOIP,CN 规则会自动处理国内域名

### 4.4 风控提示

很多金融平台有 IP 地区风控机制：
- 如果某个平台突然无法访问或提示异常，尝试切换到常用地区的节点
- 不要在短时间内频繁切换不同国家的 IP
- 建议固定使用 1-2 个地区的节点进行金融操作
