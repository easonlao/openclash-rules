# 国内、私有网络与回家场景

## 零、项目重写原则

1. **不依赖外部仓库**：所有规则在本项目内自行维护创建，不依赖第三方规则仓库。
2. **按组审查确认**：每一组规则先确认设计，再进入完整配置。
3. **最小可用原则**：只保留必要的规则，不盲目复制大量个人偏好列表。

## 一、核心背景

### 1.1 网络环境梳理

| 组件 | 说明 |
|------|------|
| **iStoreOS 软路由** | 10.10.10.20，运行 PassWall Server + Lucky 反代 |
| **PassWall Server** | 端口 12320，提供回家翻墙代理 |
| **Cloudflare DNS** | `*.l-home.top` 解析到 iStoreOS 的 IPv6 地址 |
| **公网情况** | 家里只有公网 IPv6，没有公网 IPv4 |
| **手机 Clash Mi** | Mihomo 内核，`mixed-port: 7890`，`allow-lan: false` |

### 1.2 回家场景核心问题

手机回家时如果忘记关 Clash Mi，会出现：
- 局域网访问异常（走了代理）
- 回家代理域名本身走代理，形成回环

**解决方案核心思路：私有网络 + 回家域名强制直连。**

---

## 二、最终需要产出的配置数量：2 个（不是 3 个）

| 配置 | 场景 | 核心需求 |
|------|------|----------|
| **1. 本地 OpenClash** | 家里 iStoreOS 上运行 | 家庭网关，正常分流翻墙 |
| **2. 手机统一配置** | 手机回家/外出通用 | 通过「回家开关」策略组在 `Home` 和机场节点之间切换 |

### 为什么合并？

- 不需要维护两个手机配置文件
- 回家/外出切换只需要点一下策略组
- `*.l-home.top` 直连规则和 `Home` 代理定义一直存在，不会忘
- 配置更简洁，维护成本更低

---

## 三、两个配置的共性与差异

### 3.1 基础策略组（两个配置完全相同）

见 `01-基础策略组与通用选择.md`，共用：
- 业务组默认第一项为「通用选择」
- 「通用选择」默认第一项为「所有-自动」
- 「直连」和「拒绝」放在靠后位置
- 所有 `include-all` 普通节点池提前排除 `链式代理\|住宅`

### 3.2 本分组差异对比

| 规则/模块 | 本地 OpenClash | 手机统一配置 |
|-----------|---------------|-------------|
| **IPv4 私有网络直连** | ✅ | ✅ |
| **IPv6 私有网络直连** | ✅ | ✅ |
| **GEOIP,CN 国内直连** | ✅ | ✅ |
| **`*.l-home.top` 强制直连** | ❌ | ✅（回家时不回环） |
| **回家代理 `Home` 定义** | ❌ | ✅（一直存在，切换用） |
| **「回家开关」策略组** | ❌ | ✅（在 `Home` 和机场节点间切换） |
| **`proxy-server-nameserver` 国内 DNS** | ✅ | ✅ |
| **IPv6 DNS 解析** | ✅ | ✅ |
| **自用直连规则** | ✅ | ✅ |

## 四、具体规则设计

### 4.1 私有网络直连规则（三个配置都有）

```yaml
# IPv4 私有网络
- IP-CIDR,10.0.0.0/8,DIRECT,no-resolve
- IP-CIDR,172.16.0.0/12,DIRECT,no-resolve
- IP-CIDR,192.168.0.0/16,DIRECT,no-resolve

# IPv6 私有网络
- IP-CIDR,fc00::/7,DIRECT,no-resolve
- IP-CIDR,fe80::/10,DIRECT,no-resolve

# 回环地址
- IP-CIDR,127.0.0.0/8,DIRECT,no-resolve
- IP-CIDR,::1/128,DIRECT,no-resolve
```

**关键：** `no-resolve` 参数，避免解析 IP 时走代理 DNS。

### 4.2 国内域名直连规则（三个配置都有）

```yaml
- GEOIP,CN,DIRECT
```

### 4.3 回家域名直连规则（仅手机回家配置）

```yaml
# 家里反代域名强制直连，避免回环
- DOMAIN-SUFFIX,l-home.top,DIRECT
```

### 4.4 自用直连规则（三个配置都有）

```yaml
# 自用动态域名
- DOMAIN,easonlao.ip-ddns.com,DIRECT

# m-team PT站直连
- DOMAIN-KEYWORD,m-team,DIRECT
- DOMAIN-SUFFIX,m-team.cc,DIRECT
- DOMAIN-SUFFIX,m-team.io,DIRECT
- DOMAIN-SUFFIX,manfuz.co,DIRECT
- DOMAIN,tracker.m-team.cc,DIRECT
- DOMAIN,tracker.m-team.io,DIRECT
- DOMAIN,tra1.m-team.cc,DIRECT
- DOMAIN,tra1.m-team.io,DIRECT
- DOMAIN,tra99.manfuz.co,DIRECT
```

### 4.5 回家开关策略组（仅手机统一配置）

```yaml
- name: 回家开关
  type: select
  proxies:
    - 所有-自动  # 外出时选这个，直接连机场
    - Home      # 回家时选这个，走家里 PassWall

# 通用选择里把「回家开关」放在第一项
- name: 通用选择
  type: select
  proxies:
    - 回家开关  # ✅ 优先级最高，切换回家开关就切换了所有流量的出口
    - 所有-自动
    - 香港-故转
    - 日本-故转
    - 新加坡-故转
    - 美国-故转
    - 其他-故转
    - 所有-手动
    # ... 其他节点
    - 直连
    - 拒绝
```

**使用方式**：
- **回家**：手动把「回家开关」切到 `Home`，所有流量走家里 PassWall
- **外出**：手动把「回家开关」切到 `所有-自动`，直接连机场节点

### 4.6 回家代理定义（仅手机统一配置）

```yaml
proxies:
  - name: Home
    type: ss
    server: socks.l-home.top
    port: 12320
    cipher: aes-256-gcm
    password: Yx880721
    # udp: true  # 可选，根据需要开启
```

### 4.5 DNS 配置（三个配置都有，手机回家需要特别注意）

```yaml
dns:
  enable: true
  ipv6: true  # 关键！启用 AAAA 解析，否则回家 IPv6 域名解析失败

  # 用国内 DNS 解析代理节点域名，避免鸡蛋问题
  proxy-server-nameserver:
    - https://doh.pub/dns-query

  # 其他 DNS 配置根据实际情况补充
```

**手机回家配置特别注意：**
- `ipv6: true` 必须开启，否则 `socks.l-home.top` 解析不到 IPv6 地址
- `proxy-server-nameserver` 用国内 DoH，确保能正确解析国内 IPv6 地址

## 五、规则顺序与优先级

三个配置的规则顺序建议：

```yaml
rules:
  # 1. 最高优先级：私有网络（必须第一组，避免被其他规则覆盖）
  - IP-CIDR,10.0.0.0/8,DIRECT,no-resolve
  - IP-CIDR,172.16.0.0/12,DIRECT,no-resolve
  - IP-CIDR,192.168.0.0/16,DIRECT,no-resolve
  - IP-CIDR,fc00::/7,DIRECT,no-resolve
  - IP-CIDR,fe80::/10,DIRECT,no-resolve
  - IP-CIDR,127.0.0.0/8,DIRECT,no-resolve
  - IP-CIDR,::1/128,DIRECT,no-resolve

  # 2. 回家域名（仅手机回家配置）
  - DOMAIN-SUFFIX,l-home.top,DIRECT

  # 3. 自用直连
  - DOMAIN,easonlao.ip-ddns.com,DIRECT
  - DOMAIN-KEYWORD,m-team,DIRECT
  # ... 其他 m-team 域名

  # 4. AI 服务规则（高优先级，避免分流错误导致封号）
  # 第 4 组补充

  # 5. 广告/拦截规则（第 3 组补充）

  # 6. 业务分流规则（后续分组补充）

  # 7. 国内直连
  - GEOIP,CN,DIRECT

  # 8. 兜底
  - MATCH,通用选择
```

## 六、关键注意事项

### 6.1 回家场景的常见坑

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 回家时局域网访问异常 | 私有网络 IP 走了代理 | 私有网络规则放在最前面，加 `no-resolve` |
| `socks.l-home.top` 连接失败 | DNS 没有启用 IPv6 | `ipv6: true` |
| 回家代理本身走了代理 | 回家域名被匹配到其他规则 | `*.l-home.top` 直连规则放在国内规则之前 |
| 手机切换 Wi-Fi/蜂窝后连接异常 | 网络切换导致连接状态异常 | Clash Mi 一般会自动重连，手动开关一次即可 |

### 6.2 规则优先级原则

1. **私有网络 > 特定域名 > 广告拦截 > 业务分流 > 国内 > 兜底**
2. **`no-resolve` 对 IP 规则非常重要**，避免解析 IP 时走代理 DNS
3. 回家域名规则必须放在国内规则之前，因为 `l-home.top` 是国内域名，但必须确保它直连

### 6.3 配置切换建议

手机端建议准备两个配置文件：
- **home.yaml** —— 回家时用，包含 `Home` 代理和 `*.l-home.top` 直连
- **mobile.yaml** —— 外出时用，直接连机场节点

回家时切换到 `home.yaml`，外出时切换到 `mobile.yaml`。

## 七、已确认 ✅

本分组已确认（2026-06-17）：

| # | 设计点 |
|---|---------|
| 1 | 私有网络规则放在最前面，加 `no-resolve`，避免被其他规则覆盖 |
| 2 | 手机配置加 `*.l-home.top` 直连规则，避免回环 |
| 3 | 手机配置加 `Home` SS 代理定义（`socks.l-home.top:12320`） |
| 4 | 手机配置合并为一个，通过「回家开关」策略组在 `Home` 和机场节点间切换 |
| 5 | 两个配置都开启 `ipv6: true`，确保 IPv6 地址能解析 |
| 6 | 两个配置都用 `proxy-server-nameserver` 国内 DoH，避免鸡蛋问题 |
| 7 | 最终产出两个独立配置文件：**本地 OpenClash、手机统一配置** |
| 8 | 自用直连规则：`easonlao.ip-ddns.com` + m-team 相关域名 |

### ⚠️ 特别注意事项

**AI 服务规则优先级高**：OpenAI、Anthropic 等服务对代理检测非常严格，分流错误容易导致封号。AI 服务规则将单独分组（第 4 组），放在广告规则之前，确保高优先级匹配。
