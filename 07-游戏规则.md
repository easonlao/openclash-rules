# 游戏规则

## 零、项目重写原则

1. **不依赖外部仓库**：所有规则在本项目内自行维护创建，不依赖第三方规则仓库。
2. **按组审查确认**：每一组规则先确认设计，再进入完整配置。
3. **最小可用原则**：只保留必要的规则，不盲目复制大量个人偏好列表。

---

## 一、策略组设计

### 1.1 游戏统一策略组

所有游戏相关服务统一走 `游戏` 策略组：

```yaml
proxy-groups:
  - name: 游戏
    type: select
    proxies:
      - 香港-故转      # 首选：香港节点，延迟低，适合大部分游戏
      - 所有-自动
      - 日本-故转      # 备选：日本节点，适合日服游戏
      # 其他节点...
```

**推荐默认选「香港-故转」**：物理距离近，延迟低，适合大部分在线游戏。

## 四、注意事项

### 4.1 游戏节点选择建议

| 游戏服 | 推荐节点地区 |
|--------|-------------|
| 港服 | 香港 |
| 台服 | 台湾 / 香港 |
| 日服 | 日本 / 香港 |
| 美服 | 美国 / 新加坡 |
| 欧服 | 新加坡 / 美国 |

游戏对延迟敏感，优先选物理距离近的节点。

### 4.2 UDP 支持

很多游戏使用 UDP 协议，如果你的节点支持 UDP，游戏体验会更好。
- 大部分机场节点都支持 UDP
- 如果游戏连接有问题，可以尝试切换节点或开启 UDP 转发

### 4.3 游戏更新下载

游戏更新（特别是 Steam）通常需要大量带宽：
- 可以临时切换到带宽更大的节点
- 或者使用国内镜像源（如果有的话）
- Steam 国内有 CDN，大部分情况下直连下载速度就很快

### 4.4 国服游戏注意

国内游戏（如腾讯、网易的国服）：
- **不需要走代理**，直连即可
- 走代理反而可能增加延迟，甚至被封号
- 建议添加直连规则或确保 GEOIP,CN 能够匹配到
