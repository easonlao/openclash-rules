# AI 服务规则

## 零、项目重写原则

1. **不依赖外部仓库**：所有规则在本项目内自行维护创建，不依赖第三方规则仓库。
2. **按组审查确认**：每一组规则先确认设计，再进入完整配置。
3. **最小可用原则**：只保留必要的规则，不盲目复制大量个人偏好列表。

---

## 一、核心原则：高优先级，零误判

### 1.1 为什么 AI 服务规则优先级最高

OpenAI、Anthropic 等服务对代理检测非常严格：
- **同一账号在多个地区 IP 之间频繁切换** → 封号风险高
- **国内 IP 直接访问** → 可能被限流或封号
- **分流错误导致部分请求走直连、部分走代理** → 检测风险极高

**AI 服务分流错误的后果远大于广告漏拦**，所以规则必须：
1. 放在高优先级位置（仅在私有网络、自用直连之后）
2. 覆盖所有相关域名，宁可多拦、不可漏拦
3. 明确指定走哪个地区的代理节点

### 1.2 规则位置

```yaml
rules:
  # 1. 私有网络（最高优先级）
  # 2. 回家域名（仅手机）
  # 3. 自用直连
  # 4. ✅ AI 服务规则 ← 在这里，高优先级
  # 5. 广告/拦截规则
  # 6. 业务分流
  # 7. 国内直连
  # 8. 兜底
```

## 二、策略组设计

### 2.1 AI 专用策略组

所有 AI 服务流量统一走 `AI 服务` 策略组，方便统一切换节点：

```yaml
proxy-groups:
  - name: AI 服务
    type: select
    proxies:
      - 新加坡-故转    # 首选：新加坡节点，质量更稳定
      - 新加坡-自动
      - 新加坡-手动
      - 所有-自动
      # 其他节点...
```

**推荐默认选「新加坡-故转」**：新加坡节点质量更稳定，对 OpenAI/Anthropic 等服务也很友好。

## 四、注意事项

### 4.1 IP 地区一致性

- **尽量保持 AI 服务的 IP 地区稳定**，不要频繁在不同国家之间切换
- 建议默认锁定美国节点，风险最低
- 如果遇到封号，先换一个干净的美国 IP，不要马上换其他国家

### 4.2 不要让 AI 服务走「回国代理」

回家模式下，如果 `Home` 代理是家里的国内 IP，**AI 服务不能走 `Home`**，否则 OpenAI 会检测到国内 IP，有封号风险。

**确保规则顺序正确**：AI 服务规则在 `*.l-home.top` 直连规则之后，但优先级高于兜底规则，不会被国内直连规则匹配到。

### 4.3 新增 AI 服务的原则

- 只要是需要翻墙访问的 AI 服务，都加进来
- 宁可多一条规则，不要漏一条导致分流错误
- 新服务优先测试后再正式使用，确认 IP 地区没问题

## 五、已确认 ✅

本分组已确认（2026-06-17）：

| # | 设计点 |
|---|---------|
| 1 | AI 服务规则为高优先级，放在私有网络/自用直连之后、广告拦截之前 |
| 2 | 所有 AI 服务统一走 `AI 服务` 策略组，方便切换节点 |
| 3 | `AI 服务` 策略组默认推荐新加坡节点，质量更稳定 |
| 4 | 规则覆盖：OpenAI（全家桶）、Anthropic（Claude）、**Google Gemini**、豆包、DeepSeek、智谱清言 |
| 5 | 回家模式下 AI 服务不走国内 `Home` 代理，保持走海外节点 |
