# 基础策略组与通用选择

## 目标

先解决当前模板最大的问题：所有业务组默认第一项是 `直连`。

新的结构改成：

```text
业务组 -> 通用选择 -> 实际节点组
```

这样平时只需要切 `通用选择`，个别业务需要特殊线路时再单独切对应业务组。

default: &default
  type: select
  proxies:

## 配套调整

普通节点池要排除后续可能加入的链式/住宅节点：

```yaml
filter: "^((?!(直连|拒绝|链式代理|住宅)).)*$"
```
