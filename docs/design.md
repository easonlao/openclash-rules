## 策略组完整依赖链

```
┌─────────────────────────────────────────────────────────────┐
│                        【节点层】                             │
├─────────────────────────────────────────────────────────────┤
│  聚合机场 (订阅) ← 所有节点来源，自动/手动/故转选              │
│  住宅-socks5 ← 独立SOCKS5节点，走住宅中继组当出口            │
│  直连、拒绝 ← 内置                                            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    【底层策略组（选具体节点）】                 │
├─────────────────────────────────────────────────────────────┤
│  所有-手动/自动/故转 ← 用聚合机场，不过滤                     │
│  香港-手动/自动/故转 ← 用聚合机场，过滤香港节点                │
│  台湾-手动/自动/故转 ← 同上，过滤台湾                         │
│  日本-手动/自动/故转 ← 同上，过滤日本                         │
│  新加坡-手动/自动/故转 ← 同上，过滤新加坡                      │
│  美国-手动/自动/故转 ← 同上，过滤美国                         │
│  其他-手动/自动/故转 ← 同上，排除以上地区                      │
│  住宅中继组 ← 选台湾/日本/其他-自动，给住宅-socks5当出口       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                   【中层策略组（总开关）】                      │
├─────────────────────────────────────────────────────────────┤
│  通用选择 ← 日常只改这个，选以上任何地区/所有组                │
│  回家开关 ← 手机版独有，选通用选择或回家节点                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  【业务策略组（分流目的地）】                   │
├─────────────────────────────────────────────────────────────┤
│  人工智能、流媒体、谷歌、微软、苹果、英伟达                    │
│  游戏、加密货币、测试、国外、国内、其他                        │
│  → 默认都走 回家开关/通用选择，可单独切特定节点                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    【规则集（匹配流量）】                       │
├─────────────────────────────────────────────────────────────┤
│  42 个规则集，匹配对应域名/IP，分流到上面的业务策略组           │
│  没匹配到的 → MATCH → 其他 → 通用选择                          │
└─────────────────────────────────────────────────────────────┘
```

### 关键设计说明

1. **住宅 IP 的中继链路（双层代理）**：
   ```
   流量 → 选住宅-socks5节点 → 走「住宅中继组」选的地区节点 → 住宅SOCKS5服务器 → 目标网站
   ```
   目的：隐藏真实出口，降低风控。住宅中继组目前仅保留台湾/日本/其他三个通过隔离测试的地区。

2. **分层目的**：
   - 底层 = 选具体节点（自动测速、手动选），用户日常不用改
   - 中层 = 日常总开关，只用改这个，不用管细节
   - 业务层 = 特殊分流（比如ChatGPT必须走住宅IP，Netflix必须走新加坡），默认都走通用选择，按需单独切

3. **默认流量路径**：
   99% 的流量 → 规则集匹配 → 对应业务组 → 通用选择 → 你选的默认地区节点
   只有 1% 特殊情况（比如AI服务必须住宅）才会单独切。

## 性能优化记录

### 2026-06-20 大规模去重 + mrs 化

**问题发现**：
- 多个规则集指向完全相同的 URL，同一份数据加载多份，浪费内存
- 大量 list 纯文本格式，加载慢、解析慢
- 广告规则重复 3 次（BanAD + Block + BanProgramAD），实际有效规则只有约 1/3

**优化措施**：

| 类型 | 内容 | 数量 |
|------|------|------|
| 删除完全重复 | Crunchyroll, Popcorn（与 Netflix 同 netflix.mrs） | 2 |
| 删除完全重复 | Block（与 BanAD 同 Advertising.list） | 1 |
| 删除完全重复 | Gemini（与 Google 同 google.mrs） | 1 |
| 删除完全重复 | Copilot（与 GitHub 同 github.mrs） | 1 |
| 删除子集冗余 | BanProgramAD（Advertising.list 的子集） | 1 |
| 删除功能重复 | Apple-Custome（与 Apple / Domain 重复） | 1 |
| list → mrs 升级 | Nvidia, Blizzard, EA, Epic, UBI, Nintend, PlayStation, Adobe, Claude, BanAD | 10 |

**优化成果**：
- 规则集总数：49 → 42 个（减少 7 个冗余）
- mrs 二进制：30 → 36 个（占比 61% → 86%，加载更快）
- list 纯文本：19 → 6 个（减少 13 个慢加载）
- 广告拦截：3 份冗余 ~13万条 → 1 份 category-ads.mrs 二进制

**验证脚本**：
- 9 项检查，任何错误阻止发布
- 覆盖：YAML 语法、顶级字段、重复规则集、策略组引用顺序、引用匹配、命名格式、策略组引用有效性、锚点格式一致性、URL 可访问性

## 核心原则（不对外公开）

> **IP 稳定 > 一切。宁可慢一点，也不要风控封号。**

1. **故转优先**：所有策略组默认 `type: select` 手动选择固定节点，不自动测速
2. **住宅 IP 独立**：手动配置单个节点，不是订阅链接，保证出口唯一
3. **自动测速降级**：`url-test` 仅作备选，不设为默认，避免频繁切换 IP
4. **相同服务商同一规则**，避免同一个服务商的不同域名走不同节点导致 IP 漂移
5. **住宅中继不使用美国或未经验证的自动组**

## 绝对不要做的事（不对外公开）

❌ **不要把任何策略组默认设为 url-test 自动测速** —— 5 分钟切一次 IP，任何网站都能看出来你是代理

❌ **不要用 proxy-provider 拉取住宅 IP** —— 住宅 IP 就是一条线路，不是节点池

❌ **不要频繁切换 AI 的普通/住宅出口** —— 可以手动使用链式，但应长期保持同一出口
