# 流媒体规则

## 零、项目重写原则

1. **不依赖外部仓库**：所有规则在本项目内自行维护创建，不依赖第三方规则仓库。
2. **按组审查确认**：每一组规则先确认设计，再进入完整配置。
3. **最小可用原则**：只保留必要的规则，不盲目复制大量个人偏好列表。

---

## 一、策略组设计

### 1.1 流媒体统一策略组

所有流媒体服务统一走 `流媒体` 策略组：

```yaml
proxy-groups:
  - name: 流媒体
    type: select
    proxies:
      - 新加坡-故转    # 首选：新加坡节点，对大部分流媒体友好
      - 香港-故转      # 备选：香港节点，看港台内容可能更快
      - 所有-自动
      # 其他节点...
```

**推荐默认选「新加坡-故转」**：对 Netflix、Disney+、Spotify 等主流流媒体都很友好。

## 四、注意事项

### 4.1 流媒体节点选择建议

| 服务 | 推荐节点地区 |
|------|-------------|
| Netflix | 新加坡 / 香港 / 日本 / 美国 |
| Disney+ | 新加坡 / 香港 |
| Spotify | 新加坡 / 香港 / 日本 |
| B站港澳台 | 香港 / 台湾 |

不同流媒体对不同地区解锁情况不同，可以根据实际使用调整。

### 4.2 YouTube 分组说明

当前设计中 YouTube 放在「开发服务」而不是「流媒体」，原因：
1. 很多人用 YouTube 看编程教程、技术分享，属于开发学习
2. 开发服务和流媒体默认节点都是新加坡，实际走法一样
3. 如果主要用来看视频，可以把 YouTube 域名移到「流媒体」分组

**可根据自己的使用习惯调整。**

### 4.3 流媒体测速

如果某个流媒体加载慢，可以：
1. 切换到该服务推荐的节点地区
2. 或者使用 `url-test` 类型策略组自动选最快节点
