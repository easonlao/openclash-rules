# OpenClash 规则重写

## 核心设计原则

> **IP 稳定 > 一切。宁可慢一点，也不要风控封号。**

| 原则 | 说明 |
|------|------|
| **故转优先** | 所有策略组默认 `type: select` 手动选择固定节点，不自动测速 |
| **住宅 IP 独立** | 手动配置单个节点，不是订阅链接，保证出口唯一 |
| **自动测速降级** | `url-test` 仅作备选，不设为默认，避免频繁切换 IP |
| **AI 默认独立** | 默认固定机场出口；需要时可手动选择链式开关 |
| **混合规则模式** | 自用规则手动维护（≤10条），通用规则 ACL4SSR 自动更新 |

| `config_mobile.yaml` | 手机 Clash Mi 用（含回家代理） | ~14KB | 填机场订阅链接 + 住宅IP节点信息 + Home的SS密码 |

## ⚠️ 使用前只需要填 3-4 个地方！

### 👉 机场订阅（两个配置都要填）

```yaml
proxy-providers:
  机场订阅:
    url: "https://你的机场订阅链接"  # ✅ 改这里
```

### 👉 住宅 IP 节点（两个配置都要填，手动配置，不是订阅！）

```yaml
proxies:
  - name: 住宅-socks5
    type: socks5
    server: "你的住宅IP地址"  # ✅ 改这里
    port: 12345                # ✅ 改这里
    username: "你的用户名"      # ✅ 改这里
    password: "你的密码"        # ✅ 改这里
    dialer-proxy: 住宅-中转      # 经独立机场选择组中转
```

**重要：住宅 IP 不是订阅链接！不要用 proxy-provider 拉取。**

### 👉 仅手机版：回家节点密码

```yaml
proxies:
  - name: 回家
    password: "你的SS密码"  # ✅ 改这里（回家代理的密码）
```

## ✅ 设计特点

1. **IP 稳定优先**：默认走「故转」固定节点选择，不自动测速，不频繁切换 IP
2. **住宅 IP 手动配置**：单个节点手动填，不是订阅，出口唯一
3. **几百上千条通用规则全靠社区自动维护**：通过 rule-provider 自动从 ACL4SSR 更新
4. **你只需要维护不到 10 条规则**：私有网络 + 个人域名 + m-team，一年到头也不用改
5. **按地区过滤节点**：自动从订阅里过滤「新加坡/香港/日本/美国」节点，按名字匹配
6. **AI 与 Google 独立增强**：本地版在通用模板基础上额外提供住宅 SOCKS5
7. **相同服务商在同一规则集**，避免同一个服务商的不同域名走不同节点导致 IP 漂移
8. **业务组选项统一**：普通业务只保留通用、全量故转/手动/自动和直连；地区细分集中在通用选择
9. **手机版独有回家开关 + `*.l-home.top` 直连规则**，避免回环

## 策略组层级结构

```
普通业务组
├─ 通用选择（默认）
├─ 所有-故转
├─ 所有-手动
├─ 所有-自动
└─ 直连

通用选择（本地版）
├─ 新加坡 / 香港 / 台湾 / 日本 / 美国 / 其他：故转、手动、自动
├─ 所有：故转、手动、自动
├─ 住宅-socks5
└─ 直连

人工智能 / 谷歌（本地版）
├─ 通用选择（默认）
├─ 住宅-socks5（额外选项）
├─ 所有-故转
├─ 所有-手动
├─ 所有-自动
├─ 直连
```

## 全局原则

1. **普通业务统一使用 default 模板**，切换「通用选择」即可改变默认出口。
2. **相同服务商必须在同一规则内**，避免同一个服务商的不同域名走不同节点导致 IP 漂移。
3. **本地版通用选择提供住宅 SOCKS5**；人工智能和谷歌额外把它放在一级选项，方便快速切换。
4. **住宅中继不使用美国或未经验证的自动组**；当前仅保留台湾、日本、其他三个通过隔离测试的地区自动组。
5. **手机版与家用版使用同一套通用和住宅策略**，只额外增加 `回家开关`：默认进入通用选择，需要时切到回家。

---

## 绝对不要做的事

❌ **不要把任何策略组默认设为 url-test 自动测速** —— 5 分钟切一次 IP，任何网站都能看出来你是代理

❌ **不要用 proxy-provider 拉取住宅 IP** —— 住宅 IP 就是一条线路，不是节点池

❌ **不要频繁切换 AI 的普通/住宅出口** —— 可以手动使用链式，但应长期保持同一出口

---

## 最佳实践

1. **先手动选好一个节点**，选好了就别动了，用一个月再说
2. **不要没事就切节点**，IP 越稳定，风控概率越低
3. **住宅 IP 只在必要时临时打开**，用完就关，不要一直开着
4. **AI 服务就一直用同一个地区的同一个节点**，不要换
