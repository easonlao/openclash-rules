# OpenClash 规则重写

## 目标

把当前使用的 Clash/OpenClash 模板重写成更可控的结构：

- 默认走一个“通用选择”总控。
- 国内和私有网络保持直连。
- 特殊服务可单独切换节点。
- 后续可插入链式代理开关。
- 每一组规则先确认，再进入完整配置草稿。

## 重写原则

- **不依赖外部仓库**：所有规则在本项目内自行维护创建，不依赖 liandu2024 或其他第三方规则仓库。
- **按组审查确认**：每一组规则先确认设计，再进入完整配置。
- **最小可用原则**：只保留必要的规则，不盲目复制大量个人偏好列表。

## 📦 已生成的配置文件

| 文件 | 用途 | 大小 | 你需要改的地方 |
|------|------|------|---------------|
| `config_local.yaml` | 本地软路由 OpenClash 用 | ~12KB | 填机场订阅链接 + 住宅IP订阅（如果有） |
| `config_mobile.yaml` | 手机 Clash Mi 用（含回家代理） | ~13KB | 填机场订阅链接 + 住宅IP订阅（如果有）+ Home的SS密码 |

## ⚠️ 使用前只需要填 2-3 个地方！

### 本地版 `config_local.yaml`：
```yaml
proxy-providers:
  机场订阅:
    url: "https://你的机场订阅链接"  # ✅ 改这里
  
  住宅IP:
    url: "https://你的住宅IP订阅链接"  # ✅ 有就改，没有就注释掉
```

### 手机版 `config_mobile.yaml`：
```yaml
proxies:
  - name: Home
    password: "你的SS密码"  # ✅ 改这里（回家代理的密码）

proxy-providers:
  机场订阅:
    url: "https://你的机场订阅链接"  # ✅ 改这里
  
  住宅IP:
    url: "https://你的住宅IP订阅链接"  # ✅ 有就改，没有就注释掉
```

**就这么多！不用手动填任何节点信息。**

## ✅ 设计特点

1. **你只需要填 2-3 个链接/密码**，节点自动拉取，不用手动填
2. **几百上千条通用规则全靠社区自动维护**：通过 rule-provider 自动从 ACL4SSR 更新
3. **你只需要维护不到 10 条规则**：私有网络 + 个人域名 + m-team，一年到头也不用改
4. **按地区自动选节点**：自动从订阅里过滤「新加坡/香港/日本/美国」节点，按名字匹配
5. **AI 服务完全独立策略组**，不经过链式开关，IP 稳定不漂移，避免封号
6. **相同服务商在同一规则集**，避免同一个服务商的不同域名走不同节点导致 IP 漂移
7. **全局链式开关**：一键切换是否走住宅节点多跳
8. **手机版独有回家开关 + `*.l-home.top` 直连规则**，避免回环

### 全局原则

1. **除了 AI 服务单独策略组，其他所有分组默认都走「通用选择」**，切换「通用选择」就切换了所有服务的出口。
2. **相同服务商必须在同一规则内**，避免同一个服务商的不同域名走不同节点导致IP漂移。
3. **AI 服务完全独立，不经过链式开关**，IP 稳定不漂移，避免封号风险。

### 最终产出 2 个配置文件

- **本地 OpenClash**：iStoreOS 网关用
- **手机统一配置**：回家/外出通用，通过「回家开关」策略组在 `Home` 和机场节点间切换

## 当前判断

当前模板的 `RULE-SET` 引用关系是完整的，没有发现启用规则指向不存在的 provider 或策略组。

主要需要改的是结构：

- `default` 第一项是 `直连`，会导致多数业务组默认直连。
- `国内` 继承 `default`，后续加通用选择时应拆成独立组。
- 广告/拦截规则需要自行整理维护，不依赖外部列表。
